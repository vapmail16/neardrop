# Example / reference documents

These files are **reference material** from the ContextFirst / historicals pattern (guidelines, checklists, audits). They inform how we write plans and run stage gates; they **do not** override the NearDrop repo layout, which is defined in the MVP plan and the actual monorepo (`apps/`, `packages/`, `docs/`).

| File | Use |
| ---- | --- |
| `MASTER_GUIDELINES.md` | Consolidated engineering and production best practices |
| `MASTER_CHECKLIST.md` | Review checklist pattern |
| `test-suite-system-prompt.md` | Testing assistant framing |
| `CRITICAL_E2E_AUDIT_PROMPT.md` | E2E audit prompt template |
| `HISTORICALS_CRITICAL_AUDIT_AND_PHASED_IMPLEMENTATION_PLAN.md` | Example phased plan style |

**Canonical NearDrop plan:** `docs/NEARDROP_MVP_IMPLEMENTATION_PLAN.md`.
